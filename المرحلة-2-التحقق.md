## 2. هيكل مجلد المهمة (معيار المرحلة 2)

الكود يلتزم بالهيكل التالي لكل مهمة **جديدة** (بعد تطبيق المرحلة 2):

```
jobs/{jobId}/
  ├── input.json
  ├── status.json
  ├── stage1_prd.json … stage6_blueprint.json
  ├── blueprint.json
  ├── logs.txt      ← يُنشأ فور بدء المهمة ويُحدّث عند كل تغيير حالة
  └── project/      ← يُنشأ فور بدء المهمة (فارغ حتى Sprint 3)
```

**ملاحظة:** المهمة القديمة `878db2cd` أنشئت قبل المرحلة 2، لذلك لا تحتوي على `logs.txt` ولا `project/`. أي مهمة جديدة ستضمهما.

## 3. التحقق العملي (اختياري)

لتأكيد أن المخرجات صحيحة على القرص:

1. شغّل الخادم: `cd projects/badr-forge-server && npm run dev`
2. أرسل طلباً جديداً:
   ```powershell
   Invoke-RestMethod -Uri "http://localhost:3000/api/forge/run" -Method POST -ContentType "application/json" -Body '{"projectName":"verify-phase2","description":"اختبار المرحلة 2"}'
   ```
3. افتح مجلد `jobs/{jobId}` الجديد وتأكد من:
   - وجود مجلد `project/`
   - وجود ملف `logs.txt` مع أسطر مثل `[timestamp] status: started → generating_blueprint`
   - تحديث `status.json` ثم وصول الحالة إلى `blueprint_ready`
