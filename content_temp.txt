## 📌 السياق (Context for AI Agents)

**المشروع:** بدر فورج (Badr Forge) v5.4.1
**الحالة الحالية:** واجهة مستقرة تماماً كمولد وثائق (Blueprint Generator) — مكتملة ولا تحتاج تحسينات إضافية.
**الهدف:** التحول من "مولد وثائق" إلى "مصنع مشاريع كاملة" يأخذ المدخلات وينتج مشروع Next.js/Prisma جاهز للنشر.
**البيئة:** Kwork Cloud Desktop (Linux-based cloud environment)
**الأدوات:** Claude Code, Cursor, Aider, Next.js, Vercel CLI, Git

## 🏗️ الهيكل المعماري النهائي (Target Architecture)

```
[1] واجهة Forge UI (المكتملة v5.4.1)
     ↓ POST /api/forge/run
[2] Next.js API Route (الخادم الوسيط - Backend Orchestrator)
     ↓ Claude API + API Key آمن (server-side فقط)
[3] Blueprint Generator Engine
     ↓ يحفظ forge-blueprint.json
[4] /jobs/{jobId}/ (نظام ملفات المهام)
     ↓ يشغل CLI
[5] AI Agent (Aider) — التنفيذ الآلي بدون تدخل يدوي
     ↓ يقرأ blueprint.json → يكتب الكود
[6] Auto-Fix Loop — حلقة التصحيح الذاتي
     ↓ npm run build → إصلاح → إعادة بناء
[7] Auto-Deploy — النشر التلقائي
     ↓ vercel --prod أو git push
[8] رابط حي للمستخدم ✅
```

#### المهمة 1.2: إنشاء API Route الرئيسي
**الملف:** `app/api/forge/run/route.ts`

```typescript
import { NextRequest, NextResponse } from "next/server";
import { randomUUID } from "crypto";
import { writeFileSync, mkdirSync, existsSync } from "fs";
import path from "path";

const JOBS_DIR = "/home/user/jobs";
const CLAUDE_API_KEY = process.env.CLAUDE_API_KEY!;

export async function POST(req: NextRequest) {
  const jobId = randomUUID().slice(0, 8);
  const jobDir = path.join(JOBS_DIR, jobId);

  // إنشاء مجلد المهمة
  mkdirSync(jobDir, { recursive: true });

  const body = await req.json();

  // حفظ المدخلات
  writeFileSync(
    path.join(jobDir, "input.json"),
    JSON.stringify(body, null, 2),
    "utf-8"
  );

  // تحديث حالة المهمة
  writeFileSync(
    path.join(jobDir, "status.json"),
    JSON.stringify({ status: "generating_blueprint", startedAt: new Date().toISOString() }),
    "utf-8"
  );

  // بدء التوليد في الخلفية (لا ننتظره)
  generateBlueprint(jobId, body).catch((err) => {
    writeFileSync(
      path.join(jobDir, "status.json"),
      JSON.stringify({ status: "error", error: err.message }),
      "utf-8"
    );
  });

  return NextResponse.json({ jobId, status: "started" });
}

async function generateBlueprint(jobId: string, input: any) {
  const jobDir = path.join(JOBS_DIR, jobId);
  const STAGES = [
    { name: "stage1_prd", prompt: `أنت مهندس متطلبات. أنشئ PRD كاملاً لـ: ${input.projectName}. الوصف: ${input.description}` },
    { name: "stage2_architecture", prompt: `بناءً على PRD التالي، صمم الهيكل المعماري...` },
    { name: "stage3_database", prompt: `صمم مخطط قاعدة البيانات (Prisma Schema)...` },
    { name: "stage4_api", prompt: `صمم نقاط API endpoints...` },
    { name: "stage5_ui", prompt: `صمم مكونات واجهة المستخدم...` },
    { name: "stage6_blueprint", prompt: `اجمع كل المخرجات السابقة في ملف blueprint.json واحد...` },
  ];

  const results: Record<string, any> = {};

  for (const stage of STAGES) {
    // تأخير 2 ثانية بين المراحل (Rate Limiting)
    await new Promise((r) => setTimeout(r, 2000));

    const controller = new AbortController();
    const timeout = setTimeout(() => controller.abort(), 60000);

    try {
      const res = await fetch("https://api.anthropic.com/v1/messages", {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
          "x-api-key": CLAUDE_API_KEY,
          "anthropic-version": "2023-06-01",
        },
        body: JSON.stringify({
          model: "claude-sonnet-4-20250514",
          max_tokens: 4096,
          messages: [{ role: "user", content: stage.prompt }],
        }),
        signal: controller.signal,
      });

      clearTimeout(timeout);
      const data = await res.json();
      results[stage.name] = data.content?.[0]?.text || "";

      // حفظ نتيجة كل مرحلة
      writeFileSync(
        path.join(jobDir, `${stage.name}.json`),
        JSON.stringify(results[stage.name], null, 2),
        "utf-8"
      );

      // تحديث الحالة
      writeFileSync(
        path.join(jobDir, "status.json"),
        JSON.stringify({ status: "generating", currentStage: stage.name }),
        "utf-8"
      );
    } catch (err: any) {
      clearTimeout(timeout);
      results[stage.name] = `ERROR: ${err.message}`;
    }
  }

  // حفظ البلوبرنت النهائي
  writeFileSync(
    path.join(jobDir, "blueprint.json"),
    JSON.stringify(results, null, 2),
    "utf-8"
  );

  writeFileSync(
    path.join(jobDir, "status.json"),
    JSON.stringify({ status: "blueprint_ready", completedAt: new Date().toISOString() }),
    "utf-8"
  );
}
```

#### المهمة 1.3: إنشاء Endpoint لمتابعة الحالة
**الملف:** `app/api/forge/status/[jobId]/route.ts`

```typescript
import { NextRequest, NextResponse } from "next/server";
import { readFileSync, existsSync } from "fs";
import path from "path";

const JOBS_DIR = "/home/user/jobs";

export async function GET(
  req: NextRequest,
  { params }: { params: { jobId: string } }
) {
  const statusFile = path.join(JOBS_DIR, params.jobId, "status.json");

  if (!existsSync(statusFile)) {
    return NextResponse.json({ error: "Job not found" }, { status: 404 });
  }

  const status = JSON.parse(readFileSync(statusFile, "utf-8"));
  return NextResponse.json(status);
}
```

### Sprint 4: حلقة التصحيح الذاتي (Auto-Fix Loop)
**المدة:** يوم واحد
**الأولوية:** 🟡 مهمة — هذا ما يفصل الأداة العادية عن المصنع الذكي

#### المهمة 4.1: سكربت التصحيح الذاتي
**الملف:** `scripts/auto-fix.sh`

```bash
#!/bin/bash
# حلقة التصحيح الذاتي
# الاستخدام: ./scripts/auto-fix.sh <jobId>

JOB_ID=$1
PROJECT_DIR="/home/user/jobs/$JOB_ID/project"
LOG_FILE="/home/user/jobs/$JOB_ID/logs.txt"
MAX_ATTEMPTS=5
ATTEMPT=0

cd "$PROJECT_DIR"

while [ $ATTEMPT -lt $MAX_ATTEMPTS ]; do
  ATTEMPT=$((ATTEMPT + 1))
  echo "🔄 محاولة البناء رقم $ATTEMPT من $MAX_ATTEMPTS..." | tee -a "$LOG_FILE"

  # محاولة البناء
  BUILD_OUTPUT=$(npm run build 2>&1)
  BUILD_EXIT=$?

  if [ $BUILD_EXIT -eq 0 ]; then
    echo "✅ البناء نجح في المحاولة $ATTEMPT!" | tee -a "$LOG_FILE"
    echo '{"status":"build_success","attempts":'$ATTEMPT'}' > "/home/user/jobs/$JOB_ID/status.json"
    exit 0
  fi

  echo "❌ فشل البناء. جاري الإصلاح التلقائي..." | tee -a "$LOG_FILE"

  # استخراج آخر 20 سطر من الخطأ
  ERROR_MSG=$(echo "$BUILD_OUTPUT" | tail -20)

  # إرسال الخطأ للوكيل للإصلاح
  aider \
    --message "فشل npm run build بالخطأ التالي. أصلحه:
$ERROR_MSG" \
    --yes \
    --auto-commits \
    --model claude-sonnet-4-20250514 \
    2>&1 | tee -a "$LOG_FILE"

  # انتظار قبل المحاولة التالية
  sleep 3
done

echo "❌ فشل البناء بعد $MAX_ATTEMPTS محاولات" | tee -a "$LOG_FILE"
echo '{"status":"build_failed","attempts":'$MAX_ATTEMPTS'}' > "/home/user/jobs/$JOB_ID/status.json"
exit 1
```

#!/bin/bash
# النشر التلقائي
# الاستخدام: ./scripts/deploy.sh <jobId>

JOB_ID=$1

## 🧠 الرأي النهائي المدمج (Claude + Gemini + GPT + DeepSeek)

### ما اتفقت عليه جميع النماذج بلا استثناء:

1. **واجهة v5.4.1 مكتملة — توقف عن تحسينها.** أي وقت إضافي هنا هو هدر. AbortController + Rate Limiting Delay أغلقا آخر ثغرات الاستقرار.

2. **الخطوة الوحيدة المنطقية: نقل الذكاء للخادم.** مفتاح API يجب أن يعيش server-side فقط. الواجهة ترسل الطلب وتنتظر النتيجة.

3. **نمط Job Queue بنظام الملفات** هو الأنسب لبيئة Kwork Cloud Desktop (لا حاجة لـ Redis أو قاعدة بيانات في هذه المرحلة).

4. **حلقة Build → Fix → Build** هي ما يفصل "أداة توليد" عن "مصنع حقيقي".

### أين اختلفت النماذج (وما الأفضل):

| النقطة | جيميني | جيبتي | ديب سيك | رأيي (Claude) |
|--------|--------|-------|---------|---------------|
| أداة التنفيذ | Aider أو Cline | Aider | Aider/Cline | **ابدأ بـ Aider** — أنضج وأوثق. Cline بديل لاحقاً |
| تقسيم البلوبرنت | لم يفصّل | 6 مراحل متسلسلة | مراحل مع context chaining | **6 مراحل مع تمرير السياق** — كل مرحلة تأخذ مخرجات السابقة |
| التصحيح الذاتي | ذكره كمبدأ | سكربت كل 30 ثانية | حلقة npm run build | **5 محاولات كحد أقصى** مع tail -20 للأخطاء |
| النشر | Vercel أو Git | Vercel CLI | Vercel/Git | **Vercel CLI أولاً** — أبسط وأسرع |
| الأمان | لم يفصّل | لم يفصّل | لم يفصّل | **أضف تحقق من المدخلات + rate limiting على API route** |

### ما أضيفه أنا (لم تذكره أي نماذج):

1. **Context Chaining بين المراحل:** المرحلة 2 يجب أن تستقبل مخرجات المرحلة 1 في الـ prompt — وليس فقط "بناءً على PRD". هذا يمنع التناقضات.

2. **حجم البلوبرنت:** ملف blueprint.json الضخم لن يدخل في context window الوكيل. الحل: قسّمه لملفات أصغر (database.prisma, routes.ts, components.md) واجعل الوكيل ينفذ ملفاً ملفاً.

3. **Fallback Model:** إذا فشل Claude API، جرب Sonnet بدل Opus لتقليل التكلفة. أضف try/catch مع model fallback.

4. **تنظيف المهام القديمة:** أضف cron job يحذف مجلدات /jobs/ الأقدم من 7 أيام لتوفير المساحة.

## ⚠️ تحذيرات مهمة قبل البدء

1. **لا تبدأ بكل شيء مرة واحدة.** Sprint 1 أولاً، اختبره يدوياً، ثم انتقل لـ Sprint 2.
2. **اختبر الـ API يدوياً بـ curl** قبل ربط الواجهة.
3. **Aider يحتاج مفتاح API منفصل** في متغير البيئة ANTHROPIC_API_KEY.
4. **راقب التكاليف:** كل عملية صهر = 6 طلبات API + طلبات الوكيل + طلبات التصحيح. ممكن تكون مكلفة.
5. **ابدأ بمشروع بسيط جداً كاختبار** (مثل "تطبيق قائمة مهام") قبل المشاريع المعقدة.
