# تحقق اتصال المراحل 0 و 1 و 2

## المسار الموحد (JOBS_DIR)

| المكوّن | المسار | النتيجة |
|---------|--------|---------|
| مجلد المهام | `process.cwd()/../../jobs` = `b11/jobs` | واحد في run و status |
| run/route.ts | JOBS_DIR → إنشاء jobDir، project/، logs.txt، input، status، استدعاء generateBlueprint | ✅ |
| status/[jobId]/route.ts | نفس JOBS_DIR → قراءة status.json من jobDir | ✅ |
| lib/forge-job.ts | أنواع وحالات موحّدة؛ لا مسارات | ✅ |

## تدفق الطلب

1. **POST /api/forge/run** → jobId، إنشاء مجلد المهمة + project/ + logs.txt، input.json، status = generating_blueprint، تشغيل generateBlueprint في الخلفية، إرجاع { jobId, status: "started" }.
2. **generateBlueprint** → تحديث status + logs عند كل مرحلة، حفظ stage*.json و blueprint.json، في النهاية status = blueprint_ready.
3. **GET /api/forge/status/{jobId}** → قراءة status.json من نفس المجلد، إرجاع الحالة.

المراحل 0 و 1 و 2 تستخدم نفس الهيكل ونفس المسارات؛ تم التحقق عملياً على المهمة 615805be (project/، logs.txt، status.json، blueprint_ready).

**النتيجة:** المراحل 0 و 1 و 2 تعمل معاً بشكل صحيح.
